O formulário HTML é representado pela tag de aberturaa documento HTML, sendo <form> <form/>e pode ser atribuído o atributo métado get/post e o atributo action.

Métados Get e Post

Os métodos get/post são utilizados em requisições code para enviar e receber dados entre o navegador e o servidor htpp.

get é utilizado para solicitar recursos e enviar dados visíveis na nunca use o get para enviar dados confidenciais, pois esses dados ficarão visíveis na url

post é utilizado para enviar dados de forma oculta e mais segura, pois os dados são enviados no corpo da requisição htpp, não ficando visíveis na url 

<form action="action.php" method="get"></form>
<form action="action.php" method="post"></form>

action define o local através de uma url ao qual serão enviados todos os dados recolhidos do formulário.

Target

<target> Indica que o resultado do envio do formulário será exibido em uma nova janela ou aba do navegador.

Isso é útil quando preciso abrir o resultado em uma janela separada do formulário principal.

<form action="/submit-form" method="post" target="_blank"></form>

Enctype

O atributo

<enctype> é usado para especificar como os dados do formulário devem ser codificados antes de serem enviados para o servidor. Ele é relevante apenas quando o método de envio do formulário é definido como <" method="post">Isso é necessário quando o formulário inclui um campo de arquivo input <input type="file">

  <form action="/upload" method="post" enctype="multipart/form-data">
  <label for="file">Selecione um arquivo:</label>
  <input type="file" id="file" name="file">
  <input type="submit" value="Enviar">
</form>

Label

O elemento <label> é usado para criar rótulos associados a elementos interativos, como campos de entrada input e botões. O rótulo em si não é interativo, mas quando um usuário clica no rótulo, o elemento associado recebe o foco ou ação.

O atributo for é usado para vincular o rótulo a um elemento específico na página. O valor do atributo for deve corresponder ao valor do atributo id do elemento que você deseja associar ao rótulo. Dessa forma, quando o usuário clicar no rótulo, o elemento correspondente será afetado.

<label for="nome">Nome:</label>
<input type="text" id="nome">

Quando o usuário clicar no rótulo nome o campo de entrada será selecionado automaticamente. Além disso, o elemento <label> também pode ser aninhado dentro de outro elemento, como um <button> ou uma <div>, para criar rótulos mais complexos. Por exemplo:

  
<button>
  <label for="idade">Clique para selecionar sua idade:</label>
</button>
<input type="text" id="idade">

File

As entradas de arquivo permitem que os usuários selecionem um arquivo de seu sistema de arquivos local para uso com a página atual. Se usado em conjunto com um elemento de formulário, eles podem ser usados ​​para permitir que os usuários façam upload de arquivos para um servidor.

O exemplo a seguir permite que os usuários usem a entrada de arquivo para selecionar um arquivo de seu sistema de arquivos e fazer upload desse arquivo para um script no servidor chamado upload_file.php.


<form action="upload_file.php" method="post" enctype="multipart/form-data">
   Select file to upload:
   <input type="file" name="fileSubmission" id="fileSubmission">
   <input type="submit" value="Upload your file" name="submit">
</form>

Multiple Files

Adicionando o atributo multiple o usuário poderá selecionar mais de um arquivo


<input type="file" name="fileSubmission" id="fileSubmission" multiple>

Accept Files

O atributo accept especifica os tipos de arquivos que o usuário pode selecionar. Por exemplo. .png, .gif, .jpeg.


<input type="file" name="fileSubmission" accept="image/x-png,image/gif,image/jpeg" />


  
<form>
   <input type="file" name="fileupload" accept="image/*" />
</form>

Upload de arquivos

Imagens e arquivos podem ser carregados/enviados ao servidor definindo o atributo enctype da tag de formulário para dados de formulário/partes múltiplas. enctype especifica como os dados do formulário seriam codificados durante o envio ao servidor. 

<form method="post" enctype="multipart/form-data" action="upload.php">
  <input type="file" name="pic" />
  <input type="submit" value="Upload" />
</form>

Checkbox and Radio

Atributo Descrição
type Indica o tipo de controle de entrada. Para caixa de seleção, será definido como "checkbox". Para botões de rádio, será definido como "radio".
name Usado para dar um nome ao controle, que é enviado ao servidor para ser reconhecido e obter o valor selecionado.
value O valor que será usado se a caixa de seleção ou o botão de rádio estiverem marcados.
checked Definido como "checked" se desejar que o controle esteja selecionado por padrão.

Radio

Os botões de opção são usados quando, entre muitas opções, apenas uma opção precisa ser selecionada.

radio Sim Não Talvez 

<input type="radio" value="s" name="check" > Sim
<input type="radio" value="n" name="check" > Não
<input type="radio" value="n" name="check" checked> Talvez

name especifica o nome do grupo de botões de opção, garantindo que apenas uma opção possa ser selecionada dentro do mesmo grupo.

value define o valor associado a um botão de opção. Esse valor é enviado ao servidor quando o formulário é submetido.

checked marca um botão de opção como selecionado inicialmente quando a página é carregada. Isso garante que o botão de opção com o atributo "checked" esteja inicialmente selecionado ao visualizar o formulário.

Checkbox

Check box são usada quando é necessário selecionar mais de uma opção.

Opção 1 Opção 2 Opção 3 

<input type="checkbox" name="opcao1" value="1" checked> Opção 1
<input type="checkbox" name="opcao2" value="2"> Opção 2
<input type="checkbox" name="opcao3" value="3" checked> Opção 3

Button

Existem várias maneiras em HTML para criar botões clicáveis. Você também pode criar um botão clicável usando a tag input definindo seu atributo de tipo como botão. O atributo type pode assumir os seguintes valores -

Tipo Descrição
submit Cria um botão de envio que envia o formulário quando clicado.
reset Cria um botão de redefinição que redefine os valores do formulário para seus valores iniciais quando clicado.
button Cria um botão genérico sem comportamento pré-definido que é usado para acionar um script do lado do cliente quando o usuário clica nesse botão.
image Cria um botão de imagem que pode ser clicado como um botão. 

<input type="submit" name="submit" value="Submit" />
<input type="reset" name="reset" value="Reset" />
<input type="button" name="ok" value="OK" />
<input type="image" name="imagebutton" src="" />

Hidden

Controles de formulário ocultos são usados para esconder dados na página e enviá-los ao servidor posteriormente. Eles não são visíveis para o usuário na página real.

O <input type="hidden"> um campo de entrada oculto em um formulário, permitindo inserir valores que não são exibidos, mas são enviados quando o formulário é submetido. Isso é útil para armazenar registros de banco de dados que precisam ser atualizados após o envio do formulário.


<input type="hidden" id="inputOculto" name="inputOculto" value="meuvalor">

Onclick

Os botões são normalmente usados ​​com um evento "onclick":


<input type="button" onclick="alert('hello world!')" value="Click Me">

<button type="button" onclick="alert('hello world!')">Click Me</button>

Select e Option

Atributo Descrição
name Define o nome do controle que é enviado ao servidor para ser identificado e obter o valor selecionado.
size Define o número de opções visíveis no menu suspenso. Se houver mais opções do que o tamanho especificado, uma barra de rolagem será exibida.
multiple Se definido como "multiple", permite que o usuário selecione várias opções simultaneamente.

Select

O elemento select é usado para criar um menu suspenso, onde o usuário pode selecionar uma opção entre várias. O elemento select contém um ou mais elementos option, que representam as opções disponíveis no menu.


<form>
  <label for="fruits">Escolha uma fruta:</label>
  <select id="fruits" name="fruits"> multiple
    <option value="apple">Maçã</option>
    <option value="orange" selected>Laranja</option>
    <option value="banana">Banana</option>
    <option value="grape">Uva</option>
  </select>
  <input type="submit" value="Enviar">
</form>

Option

Você pode agrupar suas opções em um menu de seleção para fornecer um layout mais estruturado em um longo lista de opções usando o elemento <optgroup>. A sintaxe é bem básica, basta usar o elemento com um atributo label para identificar o título do grupo, e contendo zero ou mais opções que deveriam estar dentro desse grupo.

Options Grupo 

<form action="">
  <fieldset>
    <legend>Options Grupo</legend>
    <select name="">
      <option value="milk">Milk</option>
      <optgroup label="Fruits">
        <option value="banana">Bananas</option>
        <option value="strawberry">Strawberries</option>
      </optgroup>
      <optgroup label="Vegetables" disabled>
        <option value="carrot">Carrots</option>
        <option value="zucchini">Zucchini</option>
      </optgroup>
    </select>
  </fieldset>
</form>

Datalist

A tag datalist especifica uma lista de opções predefinidas para um elemento input. Ele fornece um "autocompletar" recurso em elementos input. Os usuários verão uma lista suspensa de opções enquanto escrevem.

Data List 

<form action="">
  <fieldset>
    <legend>Data List</legend>
    <input list="Languages">
    <datalist id="Languages">
      <option value="PHP">
      <option value="Perl">
      <option value="Python">
      <option value="Ruby">
      <option value="C+">
    </datalist>
  </fieldset>
</form>

Match a Pattern

Para obter mais controle, use o atributo pattern para especificar qualquer expressão regular que deve ser correspondida para passar validação. Você também pode especificar um título, que é incluído na mensagem de validação se o campo não passar


<input pattern="\d*" title="Numbers only, please.">

Tag Input

Propriedade Descrição
Class Indica a classe da entrada
ID Indica o ID da entrada
Type Identifica o tipo de controle de entrada a ser exibido. Os valores aceitáveis são hidden, text, tel, url, email, senha, data, hora, número, intervalo, cor, caixa de seleção, rádio, arquivo, enviar, imagem, redefinir e botão. O padrão é texto se não for especificado, se o valor for inválido ou se o navegador não suportar o tipo especificado.
Name Indica o nome da entrada
Value Permite que o campo já contenha um valor predefinido quando a página é carregada..
Placeholder HTML5: Uma dica para o usuário sobre o que pode ser inserido no controle. O texto do espaço reservado não deve conter retornos de carro ou avanços de linha.
Required Garante que aquele campo tenha seu preenchimento obrigatório para validar e permitir o envio do formulário.
Disabled Valor booleano que indica que a entrada deve ser desabilitada. Os controles desabilitados não podem ser editados, não são enviados no envio do formulário e não podem receber o foco.
Checked Quando o valor do atributo type é radio ou checkbox, a presença deste atributo booleano indica que o controle está selecionado por padrão; caso contrário, é ignorado.
Multiple HTML5: Indica que vários arquivos ou valores podem ser passados (aplica-se apenas a entradas de tipo de arquivo e e-mail)
Autocomplete HTML5: Indica se o valor do controle pode ser preenchido automaticamente pelo navegador.
Readonly Valor booleano que indica que a entrada não é editável. Controles somente leitura ainda são enviados no envio do formulário, mas não receberão foco. HTML5: Este atributo é ignorado quando o valor do atributo type é definido como oculto, intervalo, cor, caixa de seleção, rádio, arquivo ou botão.
Alt Um texto alternativo para imagens, caso não sejam exibidas.
Autofocus O elemento <input> deve obter o foco quando a página é carregada.
Step O atributo step especifica os intervalos de números válidos. Funciona com os seguintes tipos de entrada: número, intervalo, data, data-hora-local, mês, hora e semana.
